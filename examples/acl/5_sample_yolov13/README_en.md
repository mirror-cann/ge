# Sample Usage Guide

## Description

This sample implements object detection using the YOLOv13 ONNX model.

Workflow:

1. Use `preprocess.py` to letterbox-resize images to 640x640, normalize to [0,1], and convert to .bin format.
2. Load the offline .om model and perform inference to obtain detection results (class, confidence, bbox).
3. Apply NMS (Non-Maximum Suppression) post-processing to remove duplicate detections.
4. Optionally use `draw_boxes.py` to draw detection boxes on the original image for visualization.

## Directory Structure

```
├── scripts
│   ├── build.sh                     # Build script
│   ├── run.sh                       # Run script
│   ├── preprocess.py                # Preprocessing: letterbox resize + normalize, generates .bin files
│   └── draw_boxes.py               # Visualization: draws detection results on original images
├── src
│   ├── acl.json                     # ACL initialization config file
│   ├── CMakeLists.txt               # Compilation config
│   └── sample_yolov13.cpp           # Main program
├── model                            // Model directory, create manually
│   └── yolov13.om                   // ATC-converted offline model file
├── data                             // Data directory, create manually
│   ├── bus.jpg                      // Test image
│   ├── bus.bin                      // Preprocessed .bin file generated by preprocess.py
│   └── bus_out.jpg                  // Visualization result generated by draw_boxes.py
└── CMakeLists.txt                   // Build script entry
```

## Environment Preparation

- Install CANN Toolkit and OPP operator package according to the installation guide.
- Set environment variables (adjust the path as needed):

    ```
    source /usr/local/Ascend/cann/set_env.sh
    ```

    Or for non-standard paths:

    ```
    source /home/developer/Ascend/cann-9.0.0/set_env.sh
    ```

## Implementation Steps

1. Log in to the development environment as the running user.

2. Download the code and navigate to the sample directory. "Sample directory" below refers to `examples/acl/5_sample_yolov13`.

3. Prepare the YOLOv13 model.

    1. Download the YOLOv13 weights.

        Download `yolov13n.pt` from the [YOLOv13 Release page](https://github.com/iMoonLab/yolov13/releases/tag/yolov13):

        ```
        wget https://github.com/iMoonLab/yolov13/releases/download/yolov13/yolov13n.pt
        ```

        **Note**: Please use a stable proxy for GitHub downloads. If the download is unstable, you can manually download via browser from the link above.

    2. Install YOLOv13 dependencies.

        YOLOv13 uses custom modules that the standard `ultralytics` package cannot load directly. You need to install the forked version of ultralytics from the YOLOv13 source repository:

        ```
        # Download the YOLOv13 source repository (includes forked ultralytics)
        curl -L -o yolov13.zip "https://codeload.github.com/iMoonLab/yolov13/zip/refs/heads/main"
        unzip yolov13.zip

        # Install forked ultralytics (uninstall standard version first if present)
        pip3 uninstall ultralytics -y 2>/dev/null
        pip3 install --no-deps yolov13-main/

        # Install missing dependencies
        pip3 install huggingface_hub seaborn py-cpuinfo
        ```

    3. Export to ONNX.

        ```python
        import torch
        from ultralytics import YOLO

        model = YOLO("yolov13n.pt")
        model.export(format="onnx", opset=11, imgsz=640)
        ```

        This generates `yolov13n.onnx`.

    4. Convert the ONNX model to an Ascend AI processor offline model (\*.om).

        ```
        atc --model=yolov13n.onnx --framework=5 --output=yolov13 \
            --soc_version=Ascend910A \
            --input_shape="images:1,3,640,640" \
            --output_type=FP32
        ```

        Place the generated `yolov13.om` file in the `model/` directory.

        **`--soc_version` parameter**: Specifies the Ascend AI processor model used for inference. The value must match the deployment hardware. Common ways to check:

        - **Atlas A2 training/inference series, Atlas training/inference series**: Run `npu-smi info`, prepend `Ascend` to the **Name** field (e.g., Name `910B4` → soc_version `Ascend910B4`)
        - **Atlas A3 training/inference series**: Run `npu-smi info -t board -i <id> -c <chip_id>`, combine **Chip Name** and **NPU Name** as `Chip Name_NPU Name` (e.g., Chip Name `Ascend910B4`, NPU Name `1234` → `Ascend910B4_1234`). `id` is the device ID (query via `npu-smi info -l`), `chip_id` is the chip ID (query via `npu-smi info -m`)
        - **Ascend 950 series**: Run `npu-smi info -t board -i <id>`, combine **Chip Name** and **NPU Name**. `id` is the device ID (query via `npu-smi info -l`)

        See [ATC soc_version parameter documentation](https://www.hiascend.com/document/detail/zh/CANNCommunityEdition/latest/devaids/atctool/atlasatcparam_16_0036.html) for details.

4. Prepare the test image.

    1. Download the test image from the YOLOv13 official repository and place it in the `data/` directory:

        ```
        cd data/
        wget https://raw.githubusercontent.com/iMoonLab/yolov13/main/ultralytics/assets/bus.jpg
        ```

    2. Run the preprocessing script to convert jpg to bin:

        ```
        python3 ../scripts/preprocess.py
        ```

        If you encounter "ModuleNotFoundError: No module named 'cv2'", install OpenCV as follows.

        **Note**: On aarch64 platforms (e.g., Atlas inference cards), `opencv-python` may conflict with libGLdispatch due to TLS memory allocation issues, causing `ImportError: cannot allocate memory in static TLS block`. Use `opencv-python-headless` (no GUI dependency) instead:

        ```
        pip3 install opencv-python-headless
        ```

        On x86_64 platforms or if TLS conflicts are not present, standard `opencv-python` can also be used.

## Build and Verification

1. Build:

    ```
    cd scripts/
    bash build.sh
    ```

2. Run:

    ```
    bash run.sh
    ```

3. Expected output:

    ```
    [INFO] acl init success
    [INFO] set device success
    [INFO] create context success
    [INFO] create stream success
    [INFO] load model ../model/yolov13.om success.
    ```

    ```
    cd ../data/

    ```

    This generates `bus_out.jpg` with detection boxes and labels drawn on the image.

## Notes

- Class labels use the COCO dataset 80-class labels, including person, car, bus, and other common categories.
- To process more images, modify the `inputFiles` list in the `main()` function of `sample_yolov13.cpp`.
- To adjust detection thresholds, modify `kConfThresh` (confidence threshold) and `kIouThresh` (NMS IoU threshold) in the code.
